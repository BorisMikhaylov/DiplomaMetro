### Схема связей таблиц

```
PASS_ALL
  ├── PLACE_ID         → REF_PSG_PLACES (станция / остановка / линия)
  ├── TRANSPORT_TYPE_ID → REF_TRANSPORT_TYPE (вид транспорта)

```

## 3. Качество данных

### PASS_ALL (89 млн строк, 10–16 марта 2025)

| Поле | % пустых | Проблема |
|------|----------|---------|
| `AUTH_TYPE` | ~100% | Полностью пустое — не использовать |
| `BANK_REFUSAL_CODE` | ~100% | Полностью пустое |
| `BANK_TRN_NO` | ~100% | Полностью пустое |
| `CRD_EXPIRY_DATE` | ~100% | Полностью пустое |
| `CORRECTED_TRAN_UNIQUE_ID` | ~99.8% | Только у исправленных транзакций |
| `ENTRY_DEVICE_TRAN_NO` | ~88.6% | Только при входе через конкретное устройство |
| `VEHICLE_ID` | ~87.4% | Заполнено только для наземного транспорта |
| `CPPC_VALIDATION_TYPE` | ~81.6% | Частично заполнено |
| `BUS_RT_NO` / `BUS_EXIT_NO` | ~66% | Только для наземного транспорта |
| `ROUTE_CODE` | ~9.5% | Пустой у части метро-транзакций |
| `PLACE_ID` | ~0.03% | Единичные пропуски — некритично |

**Дополнительные замечания:**
- `TRAN_DATE` у ряда строк содержит дату-время (не только дату) — поле изначально задумано как дата операции на устройстве, `INP_DATE` — время поступления в систему. Для анализа пассажиропотока использовать **`INP_DATE`** (точное время).
- Аннулирования (`TRN_TYPE_CATEGORY = 19`): коды 50, 51, 53 — нужно исключать при подсчёте реальных проходов.
- `IS_FAIL = 1` — неуспешные проходы — также фильтровать.
- `TRANSPORT_TYPE_ID` распределение (выборка 50к строк): Метро=47.7%, Автобус=23.8%, Электробус=18.4%, Трамвай=7.2%, Поезд=2.9%.

### REF_PSG_PLACES

- 3 141 запись, все `PLACE_ID` из PASS_ALL присутствуют в справочнике (100% join).
- Метро + МЦК + МЦД представлено через поле `LN_NAME` (название линии).
- **Нет координат** — `PLACE_ID` не содержит географии.
- TYPE_ID=3 (1 502 записи) — наземные остановки; TYPE_ID=1 (719) — вероятно метро-устройства; TYPE_ID=8 (576) — другие; TYPE_ID=15 (312) — виртуальные/ЖД.
- Дублирование: одна станция метро имеет несколько `PLACE_ID` (по числу турникетных стоек/входов) — при агрегации нужен GROUP BY `ST_NAME` или `ST_CODE`.

### REF_TRANSPORT_WAY

- 2 490 маршрутов: 2003 автобуса, 323 электробуса, 108 троллейбуса, 56 трамвая.
- **Метро-маршруты отсутствуют** — они вычисляются через PLACE_ID.
- Нет расписания, нет списка остановок, нет интервалов — только название маршрута.

### GDS_TRANSFER

- 3 053 узла дерева маршрутных комбинаций.
- Корни: ММ, ММТС, МЦК, МГТ, МЦД-1, МЦД-2 и пр. (дерево пересадочных цепочек).
- Дублирующихся имён нет, структура иерархическая — можно восстановить мультимодальные маршруты.

## 4. Применимость к задаче диплома

### Можно использовать сразу

| Что | Зачем |
|-----|-------|
| PASS_ALL + REF_PSG_PLACES | Расчёт пассажиропотока по каждой станции метро с разбивкой по часам — **вход в модель** |
| REF_PSG_PLACES (LN_NAME) | Перечень всех линий и станций метро |
| TRN_TYPE | Фильтрация только успешных входов (CAT=1, не аннулирование) |
| GDS_TRANSFER | Реконструкция мультимодальных маршрутов — кто пересаживался с метро на автобус |
| REF_TRANSPORT_WAY | Перечень маршрутов наземного транспорта для оценки усиления |

### Нужно подготовить / очистить

| Что | Что сделать |
|-----|-------------|
| PASS_ALL | Фильтр: `IS_FAIL=0`, `TRAN_TYPE_CATEGORY != 19`, `TRANSPORT_TYPE_ID IS NOT NULL` |
| PASS_ALL | Агрегация: группировка по `ST_CODE` (не `PLACE_ID`) + час из `INP_DATE` |
| REF_PSG_PLACES | Сопоставление дублирующихся `PLACE_ID` одной станции → единый `ST_CODE` |
| GDS_TRANSFER | Развёртка дерева в список пар «источник→цель» для маршрутного анализа |

### Чего не хватает (критические пробелы)

| Данные | Почему нужны | Возможная замена |
|--------|-------------|-----------------|
| **Координаты станций и остановок** | Расчёт пешеходных расстояний, ближайших альтернатив | Открытые данные: OSM, data.mos.ru, официальный GTFS Московского транспорта |
| **Топология метро** (граф станций с временем в пути) | Нахождение альтернативных маршрутов объезда | Открытые данные mos.ru / OSM / ручная разметка |
| **Расписание / интервалы** наземного транспорта | Расчёт вместимости и провозной способности маршрутов | GTFS Московского транспорта (transport.mos.ru) |
| **Вместимость подвижного состава** по маршрутам | Оценка перегруза при перераспределении | Нормативные данные ДТиРИ / допущения |
| **Список остановок** по маршрутам | Понять, какие автобусы проходят рядом с закрытой станцией | GTFS или data.mos.ru |

### Малополезно для задачи диплома

- `GDS_GOODS` — тарифная линейка, не влияет на потоки.
- `GDS_APPLICATION` — тип устройства (касса, АПБ и др.), не нужен для модели.
- `V_PCR_CONTRACTOR` — иерархия юридических лиц, не операционные данные.
- `AUTH_TYPE`, `BANK_*` поля — полностью пустые.

## 5. Предварительная схема модели

На основе имеющихся данных уже можно построить следующие компоненты:

### Сущности

| Сущность | Источник |
|----------|---------|
| **Станция метро** | `REF_PSG_PLACES` → `ST_CODE`, `ST_NAME`, `LN_NAME` |
| **Линия метро** | `REF_PSG_PLACES` → `LN_CODE`, `LN_NAME` |
| **Остановка наземного транспорта** | `REF_PSG_PLACES` (TYPE_ID=3) |
| **Маршрут наземного транспорта** | `REF_TRANSPORT_WAY` |
| **Пассажиропоток по станции × час** | `PASS_ALL` + агрегация |
| **Мультимодальная цепочка** | `GDS_TRANSFER` |

### Связи

- Станция → входящий/исходящий поток: GROUP BY `ST_CODE`, `HOUR(INP_DATE)`, `TRAN_TYPE_ID`
- Пересадки метро↔наземный транспорт: `GDS_TRANSFER` + `TRAN_TYPE_ID` из TRN_TYPE (коды 55, 65 = ММ→МГТ)
- Ближайшие станции / маршруты к закрытой: **требуют координат** (пока нет в данных)

### Вход модели

1. **Закрытая станция** (задаётся вручную или как параметр)
2. **Ожидаемый пассажиропоток** в период закрытия — из PASS_ALL по профилю дня/недели
3. **Список маршрутов наземного транспорта** в зоне охвата — после добавления координат

### Выход и метрики

1. **Перераспределение потока** по соседним станциям (% от исходного спроса)
2. **Нагрузка на маршруты** наземного транспорта (сравнение с расчётной вместимостью)
3. **Дефицит вместимости** — сколько пассажиров не покрыто существующими маршрутами
4. **Рекомендованные меры**: усиление (доп. рейсы), шаттл, изменение интервала
